# Первый раунд обратной связи клана

Status: awaiting feedback
Prototype stage: completed
Published site: https://svisbee.github.io/clan-analytics/

## Что уже представлено клану

- Назначение проекта.
- Публичный информационный прототип.
- Будущий состав.
- Обычные КВ.
- ЛВК.
- Рейдовые выходные.
- Пожертвования.
- Игры кланов.
- Правила приватности.
- Поэтапное развитие.

## Что пока не реализовано

- Реальные участники.
- API.
- Автоматическое обновление данных.
- SQLite.
- История КВ.
- Расчёт метрик.
- Карточки игроков.
- Внутренний интерфейс руководства.

3. Подходят ли цвета, размеры текста и расположение блоков?
4. Какие разделы наиболее полезны?
5. Что нужно убрать, добавить или переделать?
6. Какие данные должны быть в карточке игрока?
7. Нужно ли публично показывать `player_tag`?
8. Какие показатели обычных КВ наиболее важны?
9. Нужны ли рейтинги или только нейтральная статистика?
10. Нужно ли использовать название, описание, эмблему и цвета конкретного клана?

## Правила обработки

- Предложение не становится решением автоматически.
- Одинаковые предложения объединяются.
- Изменения публичных полей требуют отдельного согласования.
- Внутренние и личные данные не публикуются.
- Техническая сложность оценивается отдельно.
- Решения фиксируются до начала следующего этапа.
- Отклонённые и отложенные идеи сохраняются с причиной.
- Реальные имена и личные комментарии не добавляются в публичный документ без необходимости.

## Условия начала следующего этапа

Следующий этап начинается после:

- Получения основной обратной связи.
- Группировки предложений.
- Выбора изменений оформления.
- Утверждения публичных полей карточки.
- Решения по публикации `player_tag`.
- Получения тега клана.
- Выбора API или ручного источника состава.
