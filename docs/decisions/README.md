# Журнал решений

Для каждого архитектурного, продуктового или метрического решения создаётся отдельный Markdown-файл. Рекомендуемое имя: `YYYY-MM-DD-краткая-тема.md`.

Обязательный шаблон:

```markdown
# Название решения

- Дата:
- Вопрос:
- Принятое решение:
- Причина:
- Затронутые метрики:
- Версия логики:
- Кем согласовано:
```

Если решение изменено, старый файл сохраняется, а новое решение явно указывает заменённую версию.
