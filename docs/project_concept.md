# Концепция проекта

## Назначение

Clash Clan Analytics — система аналитики клана Clash of Clans. Планируется локальный Python-сборщик, локальная SQLite на первом этапе, расчёт показателей и статический сайт на GitHub Pages. Решение расширяется постепенно и не требует платного хостинга.

Сайт должен оставаться доступным, когда компьютер владельца выключен. Поэтому компьютер локально получает и обрабатывает данные, а GitHub Pages отдаёт заранее подготовленные статические файлы. Обновление данных происходит после локального запуска сборщика. API-токен и полная локальная база не публикуются.

## Планируемые направления анализа

- состав клана и история изменений;
- пожертвования за период;
- обычные КВ;
- атаки по зеркалу и закрытие зеркала;
- добивания и новые звёзды;
- обязательные и необязательные атаки;
- ЛВК;
- рейдовые выходные;
- Игры клана при ручном вводе;
- карточки игроков;
- очередь на следующую КВ.

## Предварительная гипотеза для обычных КВ

- Около 50% состава — проверенные игроки.
- Около 50% — новые, редко участвовавшие или нуждающиеся в повторной проверке.
- Проверенные игроки вторыми атаками страхуют проверяемую группу.
- Неиспользованная вторая атака не считается пропуском, если полезных целей уже не осталось.
- Для ЛВК правило 50/50 автоматически не применяется.

Это проектная гипотеза, а не утверждённое правило. Определения «проверенного игрока», «полезной цели» и обязательной атаки должны быть отдельно согласованы руководством клана и оформлены решением.
